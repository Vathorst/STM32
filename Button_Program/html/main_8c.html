Command Functions and Replies

Addressing Command

Format: 0 ADR x
Forward to Slave: 0 ADR x+1
Reply to Master: SLAVE x, But only if Reply to Slave didn't receive ACK.
Will copy the address specified in x into its own address buffer slave_adr.
Will then send command with x+1 to the next slave. If an ACK isn't received in time, will reply to the master instead.

Turn On Command

Format: 0 ON n
Forward to Slave: 0 ON n
Reply to Master: PRESSED x But only when the button is pressed.
Will activate the buttons and if slave_adr is the same as n will activate buzzer. Always forwards the message to the next slave. Doesn't require ACK checking so uses HAL.
Will deactivate if a message is received from the master, or send PRESSED x.

Turn Off Command

Format: 0 OFF
Forward to Slave: 0 OFF
Reply to Master : None
Will copy the message to the next slaves. Used as placeholder to turn off the slaves after the ON command has been sent.

Verify Connection Command

Format: x ASK
Forward to Slave: None
Reply to Master : ANS x
Will reply to the master personally to verify the connection.
